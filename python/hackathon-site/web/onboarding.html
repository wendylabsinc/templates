Event Onboarding

Zero to running an app on one of the event robots. By the end you'll see the event robots in the dashboard and be able to deploy and run an app on them from your terminal.

The signup site cloud.wendy.sh redirects to the canonical host cloud.wendy.dev — either link works.

1 Create your account

  1. Go to https://cloud.wendy.dev.
  2. Sign up with Google or email + password.
  3. Complete email verification if you're prompted.

You're signed in now, but you won't see any robots yet — that's expected.

2 Get added to the event organization

  1. Send the organizer the exact email address you signed up with.
  2. They'll move your account into the event organization.
  3. Once they confirm, reload the dashboard (or sign out and back in) so the organization appears — the organization list is loaded at login.

3 Verify you can see the event robots

  1. In the dashboard, open the organization switcher (left sidebar) and select the event organization.
  2. Open Assets to see the enrolled robots, or Video Streams to watch their camera feeds.
  3. If the robots are listed, you're set. (If not, see Troubleshooting.)

4 Install and authenticate the CLI

The wendy CLI is how you deploy and run apps on the robots.

Install (macOS or Linux with Homebrew):

brew install wendylabsinc/tap/wendy
wendy --version   # confirm it installed

Other platforms: download a build from github.com/wendylabsinc/wendy-agent/releases.

Authenticate against Wendy Cloud:

wendy auth login

This opens your browser. Log in with the same account from Step 1, approve the request, and return to the terminal — the CLI generates and saves its certificates automatically.

Confirm it worked and list your robots:

wendy auth status      # who you're logged in as + your organization
wendy cloud discover   # the enrolled robots you can access

New to the CLI? wendy tour is a short guided walkthrough.

5 Run an app on an event robot

From a Wendy project directory (a folder with a wendy.json — scaffold one with wendy init, or use the sample project the organizer provides):

wendy run
  • It builds your app and shows an interactive picker of the robots you can access — choose the event robot.
  • To target a specific robot and skip the picker: wendy run --device <robot-name>.
  • Handy flags: --detach (don't stream logs back), --build-type docker|swift|python, -y (auto-accept prompts).
wendy cloud run still works but is deprecated; use wendy run.

Troubleshooting

  • The event organization doesn't appear after you're added: hard-refresh, or sign out and back in.
  • The CLI says you're not authenticated, or points at the wrong place: re-run wendy auth login (you can force the host with --cloud https://cloud.wendy.dev), then check wendy auth status.
  • No robots in the picker: make sure Step 3 works first — CLI access mirrors your dashboard organization membership. If you have more than one cloud login, wendy run may ask you to pass --cloud-grpc <endpoint>.

Questions? Ask the event organizer.
